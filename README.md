# Healthetic Flutter Login Assignment

Hi! Welcome to my submission for the Healthetic Flutter Intern assignment. 

I've built a modern, smooth, and fully animated login flow. My focus was on creating a premium user experience with robust form validation and clean architecture under the hood.

Here's a breakdown of what I've included:
- A beautiful splash screen animation that transitions smoothly into the app.
- Complete Login, Sign Up, and Forgot Password screens.
- Strict form validation handling everything from empty fields to invalid emails.
- Thoughtful micro-animations for button presses, error states, and loading indicators.

## 🛠️ Prerequisites

Before you run the project, make sure you have:
- **Flutter SDK:** Version 3.0 or higher
- **Dart SDK:** ^3.8.1

## 🚀 How to Run the App

Running the app locally is super easy. Just follow these steps:

1. **Clone this repository:**
   ```bash
   git clone https://github.com/goudaritesh/-flutter-login-assignment-ritesh.git
   ```

2. **Move into the project folder:**
   ```bash
   cd healthetic-flutter-login-assignment-ritesh
   ```

3. **Grab all the dependencies:**
   ```bash
   flutter pub get
   ```

4. **Start the app!**
   ```bash
   flutter run
   ```

## 🤔 My Approach & Design Decisions

When building this, I wanted to make sure it didn't just work—it had to feel really good to use. 

* **UI/UX:** I went with a clean, vibrant aesthetic that feels modern and premium. I paid special attention to state changes, adding subtle scale and fade animations so the app feels responsive and alive, not static.
* **Code Quality:** I kept the codebase clean and modular. Screens are neatly separated in the `lib/screens/` folder, and I built reusable, custom widgets to prevent code duplication. I also created a custom `AnimatedEntrance` wrapper to make staggered animations effortless across the app.
* **Functionality:** I made sure to handle all the edge cases. The forms validate inputs correctly, the submit buttons show proper loading states while processing, and any errors are highlighted clearly in red to guide the user.
